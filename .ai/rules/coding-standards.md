# Coding Standards

1. **TypeScript Strict Mode**: No `any` types. Ensure `strict: true` is enabled.
2. **Naming Conventions**: Use `PascalCase` for Components, `camelCase` for functions, `UPPER_SNAKE_CASE` for constants.
3. **File Organization**: Co-locate tests and styles with the component/service.
4. **Error Handling**: Use custom Error classes. Don't swallow exceptions silently.
5. **Documentation**: Write JSDoc comments for public APIs and complex logic.
